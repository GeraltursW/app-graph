# 图谱用例编排 Demo

## 1. 目标

本次 Demo 将测试用例直接绑定到应用页面图谱，覆盖两类执行方式：

1. 路径采集：从根节点沿页面跳转边执行到目标页面，在目标页面采集性能。
2. 场景性能：从任意页面开始，执行滑动、长按、等待等典型操作，记录全过程性能。

页面顶部新增“图谱探索 / 用例编排”工作模式。进入用例编排后：

- 左侧展示用例目录和执行状态。
- 中间 G6 画布高亮当前用例涉及的节点与边。
- 路径边显示步骤序号和控件名称。
- 当前执行节点、边使用绿色强调，其余图谱置灰。
- 右侧展示用例配置、步骤、采集指标和模拟结果。

## 2. 终点采集：根到叶全量覆盖

终点采集不再维护少量人工示例。前端根据当前主图谱自动遍历所有非游离根节点：

```text
对每个 root 执行 DFS
  -> 找到所有没有下级跳转的 leaf
  -> 还原 root 到 leaf 的页面和边
  -> 每条路径生成一个独立用例
```

QQ 当前 300 节点 Demo 数据生成结果：

- 叶子终点：220
- 全量路径用例：220
- 最短路径：4 层
- 最长路径：5 层

其中一条路径示例：

```text
QQ 首页
  -> 消息中心
  -> 会话列表 01
  -> 聊天记录 05
  -> 群聊窗口 21
```

每一条跳转边沿用图谱中的 `widget_description`，脚本据此寻找并点击控件。每个终点都对应一条独立用例，到达后采集 CPU、内存、FPS、功耗和温度。

用例 ID 使用目标页面稳定 `page_id` 与完整路径指纹共同生成。即使未来同一终点存在多条可达路径，也会分别形成独立用例。游离节点不进入自动覆盖集，必须先确认结构或单独配置过程采集用例。

## 3. 过程采集：模板与用户配置

过程采集不预先生成所有排列组合，因为页面数、操作类型、次数、方向和时长会造成组合爆炸。当前 Demo 提供：

- 连续滑动模板
- 长按交互模板
- 混合操作模板
- 用户自定义操作序列

用户可以配置：

- 任意图谱页面作为起点
- 用例名称
- 采集时长
- CPU、内存、FPS、Jank、功耗、温度指标
- 等待、滑动、长按、点击等操作
- 操作顺序
- 滑动方向
- 重复次数
- 单次操作持续时间

创建后立即成为一条可执行的过程采集用例，并与起始页面在图谱上联动高亮。

## 4. 前端数据模型

```ts
interface GraphTestCase {
  case_id: string;
  app_name: string;
  case_name: string;
  case_type: 'path' | 'scenario';
  graph_version: string;
  start_page_id: string;
  target_page_id?: string;
  steps: TestCaseStep[];
  collection: CollectionConfig;
}

interface TestCaseStep {
  step_no: number;
  type: 'navigate' | 'tap' | 'swipe' | 'long_press' | 'wait' | 'collect';
  page_id: string;
  edge_id?: string;
  action_id?: string;
  locator?: Record<string, unknown>;
  expected_page_id?: string;
  duration_ms?: number;
  repeat?: number;
}
```

不要只保存页面标题。正式接口应保存稳定的 `page_id`、`edge_id`、`action_id` 和图谱版本，标题仅用于展示。

## 5. 推荐交互

- 左侧用例区分为“全量路径”和“过程采集”。
- 全量路径支持按终点名称搜索。
- 过程采集提供“新建”按钮和可视化配置弹窗。
- 选择路径型用例时，自动展开路径祖先节点并将路径置于视口中心。
- 非用例节点保留上下文但降低透明度，不彻底隐藏。
- 执行时按步骤逐个高亮节点和边。
- 点击步骤可以定位对应页面或跳转边。
- 失败步骤需要显示截图、当前页面识别结果、预期页面和重试记录。
- 结果区后续应增加性能时序图，并把峰值与具体步骤对齐。

## 6. 当前范围

当前版本使用前端模拟定时器和模拟性能结果，用于验证信息架构与交互。真实下发需要接入后端任务接口、设备执行脚本和性能数据流，具体契约见后端仓库：

```text
docs/test-case-orchestration-contract.md
```

当前自定义过程用例只保存在页面内存，刷新后会丢失。正式版本需通过后端用例 CRUD 持久化。

## 7. 后续计划

1. 后端根据图谱版本生成并持久化全量路径覆盖集。
2. 通过 SSE 或 WebSocket 接收实时步骤状态。
3. 将过程采集配置器接入用例 CRUD。
4. 增加性能曲线，并将时间点映射到步骤和截图。

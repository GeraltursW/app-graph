# 2026-07-14 图谱复核与演示体验迭代

## 本次目标

本轮围绕“让图谱更可信、更可读、更适合演示”完成三条能力建设：人工动作复核、截图与节点信息增强、项目宣传流程页。

## 主要改动

### 动作人工复核

- 节点动作字段统一为 `action`。
- 固定分为 `popupAction`、`stateAction`、`externalAction`、`pageNaviAction`。
- 编辑区按四个类别展示，可新增、删除和修改动作。
- 保存时整体提交 `action` 对象，人工结果覆盖 AI 判断。
- 未复核页面的跳转动作由后台根据现有边关系生成。

### 图谱与截图体验

- G6 节点直接展示截图、`page_title`、`page_text` 和 URL。
- 页面描述在节点内截断，悬浮节点展示完整信息。
- 右侧截图列表保持横向浏览。
- 同一页面存在多张截图时，全屏遮罩支持上一张、下一张和当前位置提示。

### 项目宣传页

- 右上角新增“项目旅程”入口。
- 使用 GSAP 和 ScrollTrigger 实现滚动叙事。
- 四个阶段依次展示设计方案、设备脚本、后端沉淀和前端图谱。
- 阶段文字交替从左右进入，发光曲线和流星光点随滚动连接 01 至 04。
- 宣传页使用实际 QQ 页面截图作为视觉证据。

### 演示数据

- 刷新 QQ 演示数据，保留大图谱规模。
- 首页提供多张截图用于验证同 URL 多图切换。
- 页面跳转动作由真实边关系生成，供人工复核流程演示。

## 接口思路

`GET /queryAppGraph/{app_name}` 返回每个节点的完整 `action` 对象。`POST /update_node` 使用 `multipart/form-data`，其中 `action` 是 JSON 字符串，后台校验四个数组后整体保存到 PostgreSQL JSONB 字段。

人工保存后的空数组具有业务含义，代表复核人员确认该类别没有动作，后台不能再次自动补齐。

## 验证结果

- 前端生产构建通过。
- 动作对象保存并重新查询后内容一致。
- 节点悬浮详情正常显示。
- 三张截图可在遮罩层连续切换。
- GSAP 宣传页首屏、阶段入场和滚动曲线正常工作。

## 后续计划

1. 为动作复核增加操作人、时间和修改前后差异审计。
2. 将动作目标与图谱节点做可搜索关联，减少手工填写目标 ID。
3. 按缩放等级进一步优化大图的信息密度与边标签显示。
4. 将宣传页内容接入真实项目指标和采集进度。
